# Плагины

В эту папку поместите скомпилированные .so файлы плагинов.

## Быстрый старт

1. **Поместите плагин бэкенда** (обязательно!) в `plugins/backend/`:
   ```
   plugins/backend/sfml_backend.so
   ```

2. **Запустите приложение** - плагины загрузятся автоматически

3. **Проверьте загрузку**: нажмите "Plugins" в toolbar, чтобы увидеть список загруженных плагинов

## Структура

```
plugins/
├── backend/          # Плагины графического бэкенда (DR4BackendPlugin) - ОБЯЗАТЕЛЬНО!
│   └── sfml_backend.so
└── tools/            # Плагины инструментов дорисовки (PPToolPlugin) - опционально
    └── drawing_tools.so
```

**Или просто в корень:**
```
plugins/
├── sfml_backend.so
└── drawing_tools.so
```

## Требования к плагинам

1. **Плагин бэкенда (DR4BackendPlugin)** - обязателен для работы приложения
   - Должен реализовывать интерфейс `cum::DR4BackendPlugin`
   - Должен экспортировать функцию `CreatePlugin()`
   - Пример: `sfml_backend.so`, `opengl_backend.so`

2. **Плагин инструментов (PPToolPlugin)** - опционально
   - Должен реализовывать интерфейс `cum::PPToolPlugin`
   - Должен экспортировать функцию `CreatePlugin()`
   - Пример: `drawing_tools.so`, `screenshot_tools.so`

## Загрузка

Плагины загружаются автоматически при запуске приложения из этой папки.
Или можно указать путь к плагину вручную через меню Plugins в toolbar.

## Примечание для Windows

На Windows плагины должны быть скомпилированы как .dll файлы, но стандарт использует dlopen (Linux).
Для Windows может потребоваться адаптация или использование WSL/Linux окружения.
